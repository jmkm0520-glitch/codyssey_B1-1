# 2. 시스템 설계 문서

## 2.1 문제 정의

회의록은 일반적으로 비정형 텍스트 형태로 작성되기 때문에 회의 후 핵심 의사결정 사항, 담당자, 일정, 후속 업무(Action Item)를 빠르게 파악하기 어렵다.

AI를 활용하여 회의록을 자동 분석하고 핵심 요약, 결정사항, Action Item, 담당자, 일정 정보를 구조화하여 제공하는 시스템을 설계한다.

## 2.2 업무 과업 정의

선택 과업

회의록 요약 및 Action Item 자동 추출

출력 항목

- 핵심 요약
- 결정사항
- Action Item
- 담당자
- 일정

## 2.3 타겟 사용자

- 프로젝트 매니저(PM)
- 팀장
- 서비스 기획자
- 개발팀 리더
- 운영 담당자

## 2.4 입력 데이터 형태

- 회의록 원문
- 회의 메모
- 회의 녹취록
- 협업 툴 기록(Notion, Slack, Teams)

## 2.5 출력 규격

1. 핵심 요약
2. 결정사항
3. Action Item
4. 담당자
5. 일정

# 3. 실행 로그

## Turn 1

사용자: 회의록을 정리해줘.

AI: 회의록 원문을 입력해 주세요.

## Turn 2

사용자: 모바일 앱 출시 준비 회의야.

AI: 회의록 내용을 입력해 주세요.

## Turn 3

사용자: 출시 목표일은 8월 15일이야.

AI: 결정사항으로 기록하겠습니다.

## Turn 4

사용자: UI 디자인은 현재 70% 완료 상태야.

AI: 진행 현황으로 반영하겠습니다.

## Turn 5

사용자: 박주임이 다음 주까지 최종 시안을 제출할 예정이야.

AI: Action Item으로 기록하겠습니다.

## Turn 6

사용자: QA 테스트는 이대리가 담당하고 7월 20일부터 시작해서 7월 말까지 완료 예정이야.

AI: QA 정보를 반영하겠습니다.

## Turn 7

사용자: 마케팅 예산은 최대 3000만 원이야.

AI: 마케팅 정보를 반영하겠습니다.

## Turn 8

사용자: 최종 결과를 보여줘.

AI: 현재까지 결과를 출력합니다.

## Turn 9

사용자: AWS Reserved Instance 검토도 추가해줘.

AI: Action Item에 추가하겠습니다.

## Turn 10

사용자: 결과는 표 형식으로 보여줘.

AI: 이후 결과를 표 형식으로 제공합니다.

## Turn 11

사용자: 경영진 보고용으로 작성해줘.

AI: 경영진 보고 형식으로 변경하겠습니다.

## Turn 12

사용자: 최종 결과를 다시 출력해줘.

AI: 최종 결과를 출력합니다.
